## Регрессия (Regression)

### Что это
Предсказание непрерывного числового значения на основе исторических данных.

### Простыми словами
Проводим линию через точки на графике и смотрим, что будет дальше.

### Пример из 1С
```python
# Прогноз заполнения диска
данные = [(день1, 100ГБ), (день2, 105ГБ), (день3, 111ГБ)]
# Модель: диск = 5.2 × день + 95.8
прогноз_на_30_день = 5.2 × 30 + 95.8 = 252 ГБ
```

### Алгоритмы
- Линейная регрессия (Linear Regression)
- Полиномиальная регрессия
- Random Forest Regressor
- XGBoost

### В нашем проекте
✅ **Реализовано:** `predict_disk.py` — прогноз заполнения диска

## Классификация (Classification)

### Что это
Отнесение объекта к одной из заранее известных категорий.

### Простыми словами
Как учить ребенка отличать котиков от собачек — показали много картинок, теперь он говорит "это 85% котик".

### Пример из 1С
```python
# Определение типа ошибки
ошибка = "Недостаточно прав для выполнения операции"
# Модель классифицирует:
{
    'тип': 'permission_error',
    'вероятность': 0.92,
    'рекомендация': 'Проверить настройки ролей'
}
```

### Алгоритмы
- Логистическая регрессия (Logistic Regression)
- Метод опорных векторов (SVM)
- Случайный лес (Random Forest)
- Нейронные сети

### В нашем проекте
🟡 **В планах:** `classify_user_errors.py` — классификация ошибок пользователей

## Поиск аномалий (Anomaly Detection)

### Что это
Поиск объектов, которые сильно отличаются от большинства.

### Простыми словами
Ищем "белых ворон" — то, что выбивается из общего ряда.

### Пример из 1С
```python
# Обычно в 10 утра 150 сессий
сегодня_в_10 = 40 сессий

# Правило 3-сигма
отклонение = |40 - 150| / 25 = 4.4σ  # больше 3σ → аномалия

# Модель Isolation Forest
if модель.predict(текущие_данные) == -1:
    алерт("Аномалия! Возможно, упал планировщик")
```

### Алгоритмы
- Isolation Forest
- One-Class SVM
- Правило 3-сигма (статистический метод)
- DBSCAN

### В нашем проекте
✅ **Реализовано:** `detect_anomalies.py` — поиск аномалий в сессиях

## Кластеризация (Clustering)

### Что это
Группировка похожих объектов без заранее известных категорий.

### Простыми словами
Сами раскладываем вещи по кучкам: сюда — похожие, туда — другие.

### Пример из 1С
```python
# Данные о сотрудниках
сотрудники = [
    {'документов_в_день': 78, 'ошибок': 0.5},
    {'документов_в_день': 45, 'ошибок': 1.2},
    {'документов_в_день': 23, 'ошибок': 3.5},
]

# Кластеризация K-Means
кластеры = KMeans(n_clusters=3).fit(сотрудники)

# Результат
кластер_0: "Высокая эффективность" — 15 чел.
кластер_1: "Средняя эффективность" — 45 чел.
кластер_2: "Низкая эффективность" — 8 чел.
```

### Алгоритмы
- K-Means
- DBSCAN
- Agglomerative Clustering
- Gaussian Mixture Models

### В нашем проекте
🟡 **В планах:** `analyze_user_efficiency.py` — кластеризация пользователей по эффективности

## Прогноз временных рядов (Time Series Forecasting)

### Что это
Предсказание будущих значений с учетом сезонности и трендов.

### Простыми словами
Учитываем, что по понедельникам всегда больше народу, а в выходные — меньше.

### Пример из 1С
```python
# Данные за 3 месяца по часам
нагрузка = [
    {'пн_09': 450, 'пн_10': 420, ...},
    {'вт_09': 380, 'вт_10': 410, ...},
]

# Модель Prophet
модель = Prophet(weekly_seasonality=True)
модель.fit(нагрузка)

# Прогноз на следующую неделю
прогноз = модель.predict(период=7)
print("Пик нагрузки: вторник 10:00, 470 пользователей")
```

### Алгоритмы
- Prophet (Facebook)
- ARIMA / SARIMA
- LSTM (нейросети)
- Holt-Winters

### В нашем проекте
🟡 **В планах:** `predict_server_load.py` — прогноз пиковых нагрузок

## Снижение размерности (Dimensionality Reduction)

### Что это
Сжатие данных с большим количеством признаков до малого (обычно 2-3) для визуализации.

### Простыми словами
Было 100 параметров, стало 2 — но главные закономерности сохранились.

### Пример из 1С
```python
# У каждого пользователя 50 метрик
признаки = 50

# Снижаем до 2 для графика
pca = PCA(n_components=2)
данные_2d = pca.fit_transform(все_пользователи)

# Теперь можно нарисовать
plt.scatter(данные_2d[:,0], данные_2d[:,1])
# Видим кластеры "эффективных" и "отстающих"
```

### Алгоритмы
- PCA (Principal Component Analysis)
- t-SNE
- UMAP
- Autoencoders

### Применение в 1С
- Визуализация многомерных данных
- Сжатие признаков перед кластеризацией
- Поиск скрытых зависимостей

## Рекомендательные системы (Recommender Systems)

### Что это
Предложение похожих объектов на основе истории.

### Простыми словами
"Люди, которые купили этот товар, также купили..."

### Пример из 1С
```python
# История проведения документов
история = [
    {'пользователь': 'Иванов', 'документ': 'Счет №123'},
    {'пользователь': 'Иванов', 'документ': 'Акт №456'},
    {'пользователь': 'Петров', 'документ': 'Счет №123'},
]

# Рекомендация
if пользователь открыл 'Счет №123':
    рекомендовать = ['Акт №456', 'Накладная №789']
    # "Обычно после счета оформляют акт"
```

### Алгоритмы
- Collaborative Filtering
- Matrix Factorization
- Ассоциативные правила (Apriori)
- Нейросетевые рекомендации

### Применение в 1С
- Подсказки при вводе документов
- Автозаполнение типовых проводок
- "Что делали другие в этой ситуации"

## Оптимизация (Optimization)

### Что это
Поиск наилучших параметров для достижения цели.

### Простыми словами
Перебираем варианты, но умно — не все подряд, а только перспективные.

### Пример из 1С
```python
# Настройки кластера
параметры = {
    'ras_процессов': (1, 8),
    'rmng_процессов': (2, 16),
    'rphost_процессов': (4, 32),
    'память_ГБ': (1, 16),
}

# Байесовская оптимизация
оптимум = BayesSearchCV(
    модель_производительности,
    параметры,
    n_iter=50
).fit(X, y)

print("Оптимальные настройки:", оптимум)
print("Ожидаемый прирост: +23% производительности")
```

### Алгоритмы
- Байесовская оптимизация (Bayesian Optimization)
- Genetic Algorithms
- Grid Search
- Random Search

### В нашем проекте
🟡 **В планах:** `find_optimal_settings.py` — поиск оптимальных настроек кластера

## Обработка естественного языка (NLP)

### Что это
Анализ текстов: классификация, извлечение смысла, поиск похожих.

### Простыми словами
Компьютер понимает, что написано в тексте, а не просто ищет слова.

### Пример из 1С
```python
# Текст ошибки из техжурнала
текст = "Деление на ноль при расчете себестоимости"

# Извлечение сущностей
ошибка = {
    'тип': 'деление_на_ноль',
    'операция': 'расчет_себестоимости',
    'серьезность': 'критичная'
}

# Поиск похожих ошибок
похожие = vectorizer.search(текст, history)
print("Ранее эту ошибку допускали: Иванов, Петров")
```

### Алгоритмы
- TF-IDF
- Word2Vec / BERT
- Тематическое моделирование (LDA)
- Классификация текстов

### Применение в 1С
- Классификация ошибок пользователей
- Поиск по базе знаний
- Анализ комментариев к документам
