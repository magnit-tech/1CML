## Общая информация

В этом документе собраны данные о потреблении ресурсов ML-скриптами системы 1CML. Информация основана на реальных замерах при тестировании на пилотных контурах.

### Что влияет на потребление ресурсов

| Фактор | Влияние |
|--------|---------|
| **Объем техжурнала** | Прямо пропорционально времени парсинга и размеру данных для обучения |
| **Количество пользователей** | Влияет на количество сессий и объем данных |
| **Период обучения** | 30 дней vs 90 дней — линейный рост данных |
| **Сложность модели** | Isolation Forest требует больше ресурсов, чем линейная регрессия |
| **Частота процессора** | Время выполнения обратно пропорционально частоте |
| **Количество ядер** | Парсинг и обучение масштабируются почти линейно |

## Скрипты прогноза диска

### `predict_disk.py`

**Назначение:** Ежедневный прогноз заполнения диска с помощью линейной регрессии

**Профиль нагрузки:** Низкий, кратковременный

| Параметр | Значение |
|----------|----------|
| **Частота запуска** | 1 раз в день (рекомендуется в 08:00) |
| **Длительность** | 10-30 секунд |
| **Тип нагрузки** | CPU (матричные операции) + RAM (загрузка данных) |

### Потребление ресурсов

| Компонент | Минимально | Рекомендуемо | Пик |
|-----------|------------|--------------|-----|
| **CPU (ядра)** | 0.2 ядра | 0.3-0.5 ядра | 0.8 ядра |
| **CPU (частота)** | 1.5 ГГц | 2.0+ ГГц | 2.5+ ГГц |
| **RAM** | 64 МБ | 128 МБ | 256 МБ |
| **Диск I/O** | Низкий | Низкий | Средний (при загрузке истории) |

### Детальный разбор

```python
# Данные для обучения
60 дней × 1 запись в день = 60 записей
Размер данных в памяти = 60 × 2 колонки × 8 байт ≈ 1 КБ

# Линейная регрессия
Сложность обучения = O(n × p²) где n=60, p=1
≈ 60 операций — копейки

# Прогноз
Сложность = O(1) — моментально
```

**График нагрузки по времени:**
```

 60 │       ████
 40 │   ██████████
 20 │   ██████████
  0 └─────────────────→ время
     0   10   20   30 сек
     │   │   │   │
     Загрузка Обучение Прогноз Сохранение
     данных
```

## Скрипты детектора аномалий

### `train_anomaly_detector.py`

**Назначение:** Обучение модели Isolation Forest на истории сессий (раз в неделю)

**Профиль нагрузки:** Высокий, средней длительности

| Параметр | Значение |
|----------|----------|
| **Частота запуска** | 1 раз в неделю (воскресенье в 03:00) |
| **Длительность** | 5-30 минут (зависит от объема данных) |
| **Тип нагрузки** | CPU (построение деревьев) + RAM (хранение данных) |

### Потребление ресурсов

| Компонент | Минимально | Рекомендуемо | Пик |
|-----------|------------|--------------|-----|
| **CPU (ядра)** | 2 ядра | 4-6 ядер | 8+ ядер |
| **CPU (частота)** | 2.0 ГГц | 2.5-3.0 ГГц | 3.5+ ГГц |
| **RAM** | 1 ГБ | 2-4 ГБ | 8 ГБ (при 90+ днях) |
| **Диск I/O** | Низкий | Низкий | Средний (при загрузке) |

### Детальный разбор

```python
# Данные для обучения
30 дней × 24 часа = 720 записей
15 признаков × 8 байт = 120 байт на запись
Размер данных = 720 × 120 = 86 КБ — в кэше CPU

# Isolation Forest
n_estimators = 100 деревьев
max_samples = 256 (по умолчанию)

Сложность обучения = O(n_estimators × max_samples × log(max_samples) × features)
≈ 100 × 256 × 8 × 15 = 3 072 000 операций

Время на 1 ядре 2.0 ГГц ≈ 2-3 минуты
Время на 4 ядрах 2.5 ГГц ≈ 30-45 секунд
```

**Масштабирование в зависимости от данных:**

| Период обучения | Записей | RAM | Время (1 ядро, 2.0 ГГц) | Время (4 ядра, 2.5 ГГц) |
|-----------------|---------|-----|------------------------|------------------------|
| 7 дней | 168 | 64 МБ | 30 сек | 10 сек |
| 30 дней | 720 | 256 МБ | 3 мин | 45 сек |
| 90 дней | 2160 | 768 МБ | 9 мин | 2.5 мин |
| 180 дней | 4320 | 1.5 ГБ | 18 мин | 5 мин |

```python
# Расчет для предприятия с 500 пользователями
```

```python
# 3. Использовать более простые регулярные выражения
```

## Выводы

1. **Самый тяжелый скрипт** — `techlog_parser.py`. Ему нужны SSD и 2-4 ядра CPU с частотой от 2.5 ГГц
2. **Пиковая нагрузка раз в неделю** — `train_anomaly_detector.py`. Потребляет много CPU и RAM на 5-30 минут
3. **Ежечасные скрипты** — легкие, не создают заметной нагрузки, работают на 2.0+ ГГц
4. **Частота CPU важна** — разница между 2.0 ГГц и 3.0 ГГц дает ускорение в 1.5 раза для парсинга
5. **Для старта достаточно** 4 ядер CPU 2.5 ГГц и 2 ГБ RAM под ML-скрипты (плюс отдельно под БД)

**Золотое правило:** планируйте пиковые нагрузки (обучение моделей) на ночное время, когда мало пользователей.
